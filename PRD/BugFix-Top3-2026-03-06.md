# BugFix 记录：前三项高优问题修复

- 日期：2026-03-06
- 范围：`background` / `options` / `popup`
- 目标：修复已确认的 3 个问题，并记录关键改动点，便于后续回归与审计。

---

## 1. 后台右键菜单并发创建问题（幂等 + 串行）

### 问题
- 多入口可能同时触发菜单初始化，导致重复创建同 ID 菜单项，出现不稳定行为。

### 关键改动
1. 在后台新增全局并发锁：`contextMenusSyncInFlight`。
2. `createContextMenus` 改为返回 Promise：
   - 若已有进行中的同步任务，直接复用同一个 Promise（幂等）。
   - 先 `removeAll`，再按定义顺序逐个 `create`（串行）。
3. 为 `removeAll/create` 增加 `runtime.lastError` 兜底日志，避免静默失败。
4. 删除文件底部的无条件 `createContextMenus()` 立即调用，减少冷启动重复初始化窗口。

### 影响
- 菜单初始化稳定性提升；并发触发时不会重复创建。

---

## 2. options 设置迁移误判导致无效持久化

### 问题
- 历史迁移判断中对 `listSize` 使用了不稳定比较，可能导致“无变化也写入”，引发额外存储变更事件。

### 关键改动
1. 修正 `listSizeNeedsMigration` 判断逻辑：
   - 由“规范值 vs Number(raw)”改为“规范值 vs 实际合并后的 `settings.listSize`”。
2. 增加 `settingsChanged` 深比较守卫：
   - 覆盖基础字段与对象字段（`smartTags` / `scheduledDownload` / `takeover`）。
3. 仅在“确实需要迁移且确实有变更”时才执行持久化。

### 影响
- 避免无意义写入，降低 `storage.onChanged` 噪音与后台抖动。

---

## 3. popup 智能标签域名误匹配

### 问题
- 旧逻辑使用 `includes`，会把 `evil-github.com` 误判为 `github.com`。

### 关键改动
1. 新增域名匹配函数 `matchesDomainRule(domain, ruleDomain)`：
   - 仅允许“完全相等”或“合法子域名后缀匹配（`.rule`）”。
2. 工作/社交域名匹配统一改为调用 `matchesDomainRule`。

### 影响
- 智能标签准确率提升，避免伪装域名误命中。
